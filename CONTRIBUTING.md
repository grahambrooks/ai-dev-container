# Contributing to devc

Thanks for your interest in contributing!

## Development

### Prerequisites

- Go 1.22+
- Docker
- Make

### Build

```sh
make build
```

### Test

```sh
make test
```

### Lint

```sh
make lint
```

## Releasing

Releases are automated via [GoReleaser](https://goreleaser.com/) and GitHub Actions.

### Creating a release

```sh
make release
```

This tags the current commit with a date-based version (e.g. `v2026.03.29`) and pushes the tag. The
[release workflow](.github/workflows/release.yml) then:

1. Builds `devc` binaries for Linux and macOS (amd64 + arm64)
2. Creates a GitHub Release with archives and checksums
3. Pushes an updated Homebrew formula to [grahambrooks/homebrew-tap](https://github.com/grahambrooks/homebrew-tap)

### Homebrew tap

The formula is auto-generated by GoReleaser and published to the `grahambrooks/homebrew-tap` repository. Users install
with:

```sh
brew install grahambrooks/tap/devc
```

The release workflow requires a `HOMEBREW_TAP_GITHUB_TOKEN` repository secret — a GitHub PAT with write access to the
tap repository.

## Submitting changes

1. Fork the repository
2. Create a feature branch (`git checkout -b feature/my-change`)
3. Make your changes and add tests
4. Run `make test && make lint` to verify
5. Commit with a clear message describing what and why
6. Open a pull request

## Reporting issues

Open an issue on GitHub with:

- What you expected to happen
- What actually happened
- Steps to reproduce
- `devc` version and OS
